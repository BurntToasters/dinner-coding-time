3.7.10 Comparing Circles

BTW

Because of some obscure features of the Java compiler, this assignment will work with == or .equals(). You don't need to change anything for this one! Even so, you should still use the code below. This trick will fail on more complex assignments, where it isn't as easy for the compiler to optimize.

public class CircleTester { public static void main(String[] args) { Circle one = new Circle(10, "blue", 50, 35); Circle two = new Circle(10, "blue", 50, 35); Circle three = new Circle(20, "red", 0, 0); Circle four = three; // Modify this program to correctly compare objects // We should not be comparing objects using == if(one.equals(two)) { System.out.println("Circles one and two are equal!"); System.out.println(one); System.out.println(two); } if(three.equals(four)) { System.out.println("Circles three and four are equal!"); System.out.println(three); System.out.println(four); } } }
